# VAST 模型分工路由

## 分工表

| 职责 | 模型 | 触发条件 |
|------|------|----------|
| 日常问答 / 百科 | claude-haiku-4-5 | 默认对话，简单问题 |
| 写作 / 文案 | GPT-4.1 | "帮我写"、"修改"、"润色" |
| 深度推理 / 复杂任务 + N8N | claude-sonnet-4-6（最强配置） | 复杂分析、多步推理、工作流编排 |
| 超强推理链 | DeepSeek R1 V3 | 需要完整推理过程、数学、逻辑链 |
| 知识增强 / 多模态 | Google Gemini 2.5 Pro | 联网搜索、长文档、图文混合 |
| 画图 / P图 / 图像处理 | Google Imagen 3 | "画"、"生成图"、"图片"、"修图"、"抠图" |
| 搜索增强 | SearXNG + haiku | 不确定的事实 → 先搜再答 |
| 记忆读写 | haiku（看管） | 每轮对话自动触发，后台静默 |

## 三端模型分配

| 产品端 | 核心模型 | 说明 |
|--------|----------|------|
| **CLI** | claude-sonnet-4-6 | 推理+代码，VAST最强配置 |
| **IDE** | GPT-4.1-mini + OpenHands | GPT驱动够用，OpenHands执行代码任务，成本低 |
| **APP** | 按模块路由（见上表） | 画图/推理/百科各用专属模型 |

## 成本原则
- VAST 最强模型上限 = claude-sonnet-4-6，不用 Opus
- 画图专用 Google Imagen，不走 Claude
- 推理链用 R1 V3，成本低且透明
- 日常用 haiku，能解决的不升级
- 能用便宜模型解决的，绝不用贵的

## 升级规则
- haiku 遇到超出能力的 → 自动升 sonnet
- sonnet 遇到需要深度推理的 → 自动升 opus
- 升级对用户透明，不需要告知

## 记忆系统
- 短期：当前对话上下文（自动）
- 长期：Qdrant 向量库（按项目分库）
- 坑记录：永不删除
- 套路记录：满了淘汰最旧的
- 写入看管：haiku 过滤噪音，不是什么都存
