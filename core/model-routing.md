# VAST 模型分工路由

## 分工表

| 职责 | 模型 | 触发条件 |
|------|------|----------|
| 日常问答 / 百科 | claude-haiku-4-5 | 默认对话，简单问题 |
| 写作 / 文案 | claude-sonnet-4-6 | "帮我写"、"修改"、"润色" |
| 深度推理 / 复杂任务 | claude-opus-4-7 | 复杂分析、多步骤任务、不确定时升级 |
| 画图 | flux / dall-e-3 | "画"、"生成图"、"图片" |
| 搜索增强 | SearXNG + haiku | 不确定的事实 → 先搜再答 |
| 记忆读写 | haiku（看管） | 每轮对话自动触发，后台静默 |
| 工作流推理 | N8N + sonnet | "自动化"、"每天"、"定时"、"流程" |

## 升级规则
- haiku 遇到超出能力的 → 自动升 sonnet
- sonnet 遇到需要深度推理的 → 自动升 opus
- 升级对用户透明，不需要告知

## 记忆系统
- 短期：当前对话上下文（自动）
- 长期：Qdrant 向量库（按项目分库）
- 坑记录：永不删除
- 套路记录：满了淘汰最旧的
- 写入看管：haiku 过滤噪音，不是什么都存
